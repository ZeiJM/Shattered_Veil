SECTION D — RANGE TAG LANGUAGE STANDARDIZATION
==================================================

19. Standardize all player-facing range labels.

All character, class, skill, enemy, and battle UI range labels should use Shattered Veil’s own range language.

Use these player-facing range tags only:
- Close Range
- Mid Range
- Long Range

Do not use “melee” as a player-facing range tag.
Do not use “ranged” as a player-facing range tag.
Do not use “projectile” as a player-facing range tag unless describing a specific visual effect, not a range category.

Range tag definitions:
- Close Range: short-distance attacks and abilities, usually adjacent or near-adjacent tiles.
- Mid Range: flexible skills that can target several tiles away.
- Long Range: far-reaching skills, beams, shots, rituals, or large-distance abilities.

20. Add a display helper for range tags.

Add a helper if needed:
getDisplayRangeTag(skillOrAction)

Suggested mapping:
- range 0–1 = Close Range
- range 2–4 = Mid Range
- range 5+ = Long Range
- self-only actions = Self
- global/arena-wide effects = Field / Arena-Wide, only for Veilbreaks, fields, or boss mechanics

Important:
- “Self” and “Arena-Wide” are allowed special cases.
- Normal class/skill range identity should still use Close Range, Mid Range, or Long Range.
- Do not break range calculations. This is mainly a player-facing wording and UI clarity change.

21. Update UI text to use the new range tags.

Update:
- battle action cards
- skill tooltips
- class summaries
- enemy/boss skill cards where range labels appear
- manual/help text if it uses melee/ranged labels
- any range badges in the battle UI

If internal code uses variables like melee/ranged and renaming them is risky:
- keep internal variable names for now
- convert them through getDisplayRangeTag()
