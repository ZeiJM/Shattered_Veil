## High-Risk Assets That Should Not Be Deleted Yet

- Any asset under `public/` or `src/` that appears unreferenced but may be loaded dynamically.
- Any music/audio file until the intended high-quality audio route is verified.
- Any map, class, character, terrain, or battle image until screenshots confirm it is obsolete.
- Any asset referenced by generated data, saves, localStorage, or server payloads.

## Recommended Next Cleanup Steps

1. Run `npm run audit:assets`.
2. Manually review possibly unreferenced assets before deletion.
3. Prioritize large referenced assets for compression or modern format replacement only after visual comparison.
4. Replace old/low/placeholder assets through explicit manifest routing first, not by deleting files.
5. Keep deletions in separate PRs with screenshots and rollback notes.
